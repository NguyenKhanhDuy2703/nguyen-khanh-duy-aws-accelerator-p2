T2 — D1

GitOps & CI/CD

Tập trung vào workflow, sync Git → cluster, và rollback đúng quy trình GitOps.

Task list

15 task theo ngày T2

1. Fork repo mẫu, tạo nhánh feat/hello-gitops bằng git checkout -b feat/hello-gitops.
2. Viết workflow GitHub Actions để chạy plan khi mở PR và apply khi merge vào main. File: .github/workflows/gitops.yml.
3. Cài ArgoCD vào Minikube, tạo Application trỏ tới repo GitOps của bạn bằng kubectl apply -n argocd -f install.yaml.
4. So sánh rollback bằng git revert và kubectl rollout undo; ghi kết quả vào reflection.md.
5. Commit hàng ngày với message [W9-D1] gitops: setup argocd app.
Tip: Bắt đầu với App of Apps: tạo Application cha quản lý nhiều Application con, sync 1 app trước rồi mới nâng cấp.

T3 — D2

Observability & SLO

Mục tiêu: đo lường, cảnh báo và dashboard hóa service bằng Prometheus, Grafana, Loki.

Task list

Thực hành T3

6. Deploy Prometheus + Grafana + Loki lên Minikube bằng Helm.
7. Định nghĩa 2 SLI: availability (%) và latency p99 (ms).
8. Viết PromQL query đo error rate: rate(http_requests_total{status=~"5.."}[5m]) / rate(http_requests_total[5m]).
9. Tạo alert multi-window burn rate với fast window 1h × 5m, slow window 6h × 30m.
10. Import dashboard Grafana, thêm panel SLO burn rate và chụp screenshot vào reflection.md.
Tip: Bắt đầu bằng OTel Collector nhận metric từ app rồi đẩy vào Prometheus; dùng Helm chart opentelemetry-collector nếu muốn nhanh hơn.
T3 — Observability & SLO
04
T4 — D3

Canary & Progressive Delivery

Kiểm thử deploy an toàn bằng canary, analysis template và tự động abort / rollback khi sai.

Task list

Thực hành T4

11. Cài Argo Rollouts controller và kubectl plugin vào Minikube.
12. Chuyển Deployment W8 thành Rollout CRD, dùng strategy canary 10% → 30% → 100%.
13. Viết AnalysisTemplate dùng Prometheus query, abort khi error rate > 5%.
14. Dùng k6 inject lỗi giả (ví dụ 50% error rate) để quan sát Rollout tự abort.
15. Ghi toàn bộ flow vào reflection.md: GitOps sync → deploy canary → alert → abort.
Tip: Test abort trước buổi live 15h. Nếu Rollout stuck ở trạng thái Degraded, dùng kubectl argo rollouts get rollout --watch để debug.
