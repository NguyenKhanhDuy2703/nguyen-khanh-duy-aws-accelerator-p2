## 4. Scaling Test

### 4.1 Scale Up (2 → 4 replicas)

```bash
kubectl scale deployment my-app-deployment --replicas=4
```

- 2 Pod mới: `ContainerCreating` → `Running` trong **~13 giây**
- Zero downtime trong suốt quá trình

### 4.2 Scale Down (4 → 1 replica)

```bash
kubectl scale deployment my-app-deployment --replicas=1
```

- 3 Pods `Terminating` song song

## 6. Issues & Fixes

| Issue | Root Cause | Fix |
|---|---|---|
| `/home/ubuntu/.kube/config not found` | `sudo -i -u ubuntu` tạo shell mới, không share env | Đổi sang `export HOME=/home/ubuntu` trong cùng SSH session |
| `502 Bad Gateway` | `deployment.yaml` có `targetPort: 3000` nhưng app listen port 80 | Fix `containerPort: 80` và `targetPort: 80` |
| State lock stuck | Terraform apply bị interrupt, DynamoDB item bị corrupt JSON | `aws dynamodb delete-item` xóa trực tiếp |
| ENI orphaned, subnet không xóa được | Minikube Docker bridge network chưa cleanup khi EC2 terminate | `clean_enis.ps1`: SSH → `minikube delete` → terminate orphaned instances |
| Destroy stuck 4+ phút | 3 EC2 cũ từ failed applies vẫn running, giữ ENI | Terminate 3 orphaned instances qua AWS CLI |
